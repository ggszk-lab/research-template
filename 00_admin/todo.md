## 使い方

- このファイルが「次に何をするか」の**唯一の入口**。各項目は **1行＋詳細リンク**で書く。
- 詳細は各リンク先（`00_context/decisions.md` / `04_docs/` の各ノート / `03_results/` 等）が正。ここは**状態の一覧**であり、内容を二重管理しない。
- タスクは課題管理ツール（GitHub issues 等）に分散させず、本ファイルに追記して一本化する。
- 各タスクに `[auto:: yes]`（AI 自律実行可: 実験・分析・調査・ドラフト）/ `[auto:: no]`（人間タスク: 判断・メール・提出操作）を付ける。Obsidian の横断ダッシュボード（`Planning/research/_dashboard.md`）がこのタグを読んで作業を派遣する。
- 研究の節目（中間確認・提出前）の確認は `00_admin/checklist.md`。
- 更新は `/update-todo` で git log・`decisions.md` と突き合わせられる。
