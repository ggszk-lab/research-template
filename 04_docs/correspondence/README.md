# correspondence — 外部やり取りの送信ログ

投稿・presubmission inquiry・編集部/共同研究者との重要メールなど、**外部とのやり取りの事実**を
記録する場所。`00_context/decisions.md`（判断）と対で使う。

## 運用ルール

- **発生したその場で書く**（後でまとめない。記録がないと「送ったか不明」状態が生じる）
- 1やり取り（スレッド）= 1ファイル: `<用途>_<YYYY-MM-DD>.md`（日付は起点＝初回送付日）
  - 例: `emr_presubmission_inquiry_2026-07-04.md` / `apsipa_submission_2026-07-09.md`
- 返信・追送はファイルを分けず、同ファイルの送信ログと「返信要旨」節に追記する
- 投稿システムでの提出も対象（メールでなくても、整理番号・Manuscript ID・提出日時を記録）
- decisions.md からは「関連:」でこのファイルを指す。判断の理由はあちら、事実の経過はこちら

## 雛形

```markdown
# <用途>（YYYY-MM-DD）

## 目的

（何を確認/依頼/提出するのか。1〜3行）

## 送付先・送付方法

- **To**: （宛先アドレス。公式ページ等での確認元も書く）
- **Cc/From**: （From は所属アドレスを原則とする）
- **Subject**: `...`

## 本文

（送付した最終版の全文。改稿した場合は最終版のみ残し、旧版は履歴に任せる）

## 送信ログ

- [ ] **YYYY-MM-DD 送信済み**（To/Cc/From を確定値で記録）
- [ ] 返信受領（要旨を下の「返信要旨」節に追記 → 関連ノート・venue ノートを更新）
- [ ] 無返信 N 週間（〜YYYY-MM-DD 目安）→ そのときの対処方針をここに書いておく

## 返信要旨（YYYY-MM-DD 受領）

（返信の要点。判断が必要になったら decisions.md にエントリを立て、ここからリンク）
```
