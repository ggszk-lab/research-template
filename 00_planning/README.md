# 00_planning/

プロジェクト固有の **計画・構想ノート** を置く場所です。研究の出発点となるアイデア検討・問題の定式化・実験計画・サーベイの下書き・次の方向性など、「これから何をするか」を考えるための文書を蓄積します。

## このフォルダの位置づけ

- `00_context/` が **確定した前提**（目的・RQ・制約・判断ログ）を保持するのに対し、`00_planning/` は **まだ動いている思考**（構想・計画・検討中の方向性）を保持します。
- 内容が固まったら `00_context/context.md` や `decisions.md` へ昇格させ、ここには「考えている最中のもの」を残します。

## Obsidian を起点にする運用（任意）

研究はしばしば Obsidian（Vault）側でのアイデア検討から始まり、実験が始められそうになった段階で repo が作られます。この流れに合わせて、**repo ができた時点で Vault 上のプロジェクト固有の計画ノートを `00_planning/` へ「卒業」させ、git 管理下に置く** ことを推奨します。

- **正本 = repo、Obsidian = ビューア**。Vault 側は `04_docs/literature` や `04_docs/paper` と同じ向きの **フォルダ symlink**（`<vault>/.../<project>/planning -> <repo>/00_planning`）に置き換えます。
- **横断ファイルは卒業させない。** 複数プロジェクト共有・Vault レベルのノート（投稿戦略・venue 一覧・助成金メモ等）は Vault 側に残します。
- 卒業でノートの階層が 1 段深くなると、Vault 内の他ノートへの相対リンクが切れることがあります。横断ファイルへの参照は深さ非依存の **Obsidian wikilink `[[note-name]]`** に変換しておくと安全です。
