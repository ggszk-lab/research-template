`00_admin/todo.md` を最新の作業状況に合わせて更新してください。

手順：

1. `00_admin/todo.md` を読み、現在の項目とセクション構成を把握する
2. 直近の git log（`git log --oneline -20`）と `00_context/decisions.md` の末尾を読む
3. 突き合わせて、以下の更新案を作る：
   - **完了した項目** — git log やコミット内容から完了が読み取れる「次の一手」「トラック」内の項目を「完了ログ」へ移す
   - **新規タスク** — `decisions.md` の最近の判断から派生した未着手タスクがあれば、該当セクションに1行＋詳細リンクで追記する
   - **二重管理の点検** — 各項目が「1行＋詳細リンク」に収まっているか確認し、本文に詳細が膨らんでいればリンク先へ寄せる
   - **古い項目** — 文脈が失われた・もう着手しない項目があれば指摘する（勝手に消さない）
4. ファイル冒頭の「最終更新」を今日の日付（絶対日付）に更新する
5. 更新内容を提示し、書き込み前に確認を求める

判断に迷う項目は推測で動かさず、ユーザーに確認してください。
